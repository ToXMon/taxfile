TASK: Server-Side OCR Fallback — sharp + tesseract
FILES: taxfile/src/modules/document-extraction/ocr/server-ocr.ts
DEPENDS ON: 0011
COMPLEXITY: medium

DESCRIPTION:
Server-side OCR fallback: sharp for preprocessing (grayscale, contrast, deskew), tesseract for extraction. Same OCRResult type as client.

APPROACH:
sharp preprocessing pipeline. tesseract.js Node API. Same output shape as client-ocr.

OUTCOME:
Server-side OCR with preprocessing.

ACCEPTANCE CRITERIA:
- Same OCRResult type as client-ocr.ts
- Image preprocessing applied
- Accepts Buffer + MIME
- Under 300 lines

DECISIONS LOG:
- Server-side tesseract worker created/terminated per-page (not singleton like client) — simpler in Node.js, no browser lifecycle concerns
- PDF conversion uses pdfjs-dist/legacy/build/pdf.mjs (legacy build) for Node.js compatibility vs client-ocr which uses default build
- PDF rendering uses node-canvas package (imported as 'canvas') — required for server-side canvas
- Preprocessing pipeline: grayscale → normalize → linear contrast → sharpen → PNG output
- Default contrast 130 (slight boost) chosen as safe default for tax documents
- PreprocessOptions exposed as parameter for caller customization
- Same OCRResult output shape as client-ocr.ts for seamless fallback switching
- 143 lines, well under 300 limit
- ACCEPTED: all criteria met
