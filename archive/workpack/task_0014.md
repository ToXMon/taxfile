TASK: W-2 Field Mapper
FILES: taxfile/src/modules/document-extraction/mappers/w2-mapper.ts
DEPENDS ON: 0012
COMPLEXITY: large

DESCRIPTION:
Map OCR text to W-2 fields: EIN, employer info, SSN (masked), wages (box 1), federal tax (box 2), SS wages (box 3), SS tax (box 4), Medicare (boxes 5-6), state (boxes 16-19). Regex patterns + positional heuristics. Handle OCR errors.

APPROACH:
W2Field enum. Regex per field. TaxLineItem with source.formType=W2 and boxNumber. Confidence from regex match. OCR error correction (O->0, l->1).

OUTCOME:
W-2 mapper producing structured ExtractedDocument.

ACCEPTANCE CRITERIA:
- Maps boxes 1,2,3,4,5,6,16,17,18,19
- Each field is TaxLineItem with source tracing
- SSN masked
- Confidence per field
- Under 300 lines

DECISIONS LOG:
- W2FieldDef.isDollar field defined but not yet used in logic — kept for future extensibility (e.g., non-dollar fields like employer name)
- SSN stored with value:0 in TaxLineItem since SSN is not a numeric tax value — the masked SSN is in the label string
- EIN similarly stored with value:0, actual EIN in label
- OCR error correction context-aware: O→0 and l→1 only when adjacent to digit/decimal/dollar/comma characters
- Confidence boost of +0.15 for successful regex match on top of base OCR confidence
- SSN/EIN confidence set to fixed values (0.85/0.80) since format validation is binary
- uuid used for document ID generation — needs uuid package installed
- 172 lines, well under 300 limit
- ACCEPTED: all criteria met
